# ADR 0001：文件存储决策

## 状态

已采纳。

## 背景

当前应用的核心数据包括文章、导航、站点设置和运行时配置。数据规模较小，写入入口集中在编辑后台，部署形态以单实例 Docker 为主。

阶段零的真正风险不是存储模型本身，而是默认零备份、R2 明文备份和备份队列掉电丢失。直接切换数据库会扩大迁移面，但不能自动解决远端灾备和恢复一致性问题。

## 决策

当前版本继续使用运行时文件存储，并强化以下能力：

- 原子写入和恢复前备份。
- 备份前后关键文件 fsync。
- R2 远端备份加密。
- 备份任务落盘和启动补偿 drain。
- 首次启动必须配置 R2，或主动确认跳过风险。

## 后果

文件存储保持部署简单，不引入数据库运行依赖。代价是仍然不适合多实例同时写入，后续如果需要水平扩展或复杂查询，需要新增数据库 ADR。

## 替代方案

直接迁移数据库可以改善多实例写入和查询能力，但需要迁移工具、备份策略、运维依赖和测试覆盖同步升级。现阶段收益不覆盖阶段零数据容灾风险。
