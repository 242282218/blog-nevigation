## 实施建议

### 优先级铁律
**数据丢失风险 > 发布可靠性 > 安全自动化 > 体验 > 架构天花板**。任何时候资源受限优先保障阶段零、一，阶段四仅按需启动。

### 并行策略
- 阶段零必须串行（数据容灾互相依赖）；
- 阶段一可拆分两路并行：CI/CD 修复 + 测试修复；
- 阶段二前端优化各任务可并行（移动端/暗色/编辑器/一致性）；
- 阶段三架构与测试可并行。

### 风险控制
- 每阶段末验证列表必须 100% 通过再进入下一阶段；
- 关键改动（存储层/锁/备份）必须有回退计划且部署前灾备演练；
- 阶段四任务仅在出现真实需求时启动，不为假设场景过度设计。

### 成功标准（全程）
- P0/P1 问题修复率 100%；
- 每阶段 `npm run check` + CI 全绿；
- 代码审查覆盖所有安全敏感改动（存储/备份/鉴权/部署）；
- 季度灾备演练通过率 100%；
- 用户无感知数据丢失事件 = 0。
