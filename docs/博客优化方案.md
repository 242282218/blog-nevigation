# Blog 优化方案

> 基于代码审查 + 对标 leerob.io / antfu.me / paco.me / overreacted.io 生成。
> 每条优化均标注优先级（P0 必做 / P1 重要 / P2 锦上添花）和预估工作量。

---

## 一、首页（`src/app/page.tsx`）

### 1.1 Hero 区域信息密度过高 [P0]

**现状**：Hero 左侧文字 + 右侧 intro card，再下方紧跟 3 张 writing topics 卡片，视觉重心分散。

**对标**：leerob.io / antfu.me 首页 Hero 极简，一句话定位 + 一个 CTA，留白充足，让访客快速抓住核心。

**改法**：
- 删除 `writingTopics` 静态卡片区块（内容与 `/blog` 页重复，无增量信息）
- Hero 右侧 intro card 改为可选展示（通过 settings 控制 `showIntroCard: boolean`）
- Hero 描述文字 `heroDescription` 限制在 2 行以内（CSS `line-clamp-2` 或字数校验）

```
// src/app/page.tsx — 删除整个 writingTopics section（约第 142-163 行）
```

---

### 1.2 首页缺少阅读进度感知 [P1]

**现状**：最近笔记列表只展示 4 篇，无任何"还有多少篇"的提示，访客不知道内容体量。

**改法**：在"最近整理的笔记"标题旁加总数角标：

```tsx
// 现有代码约第 167-176 行
<h2 className="mt-1 text-2xl font-semibold text-fg">最近整理的笔记</h2>
// 改为：
<h2 className="mt-1 text-2xl font-semibold text-fg">
  最近整理的笔记
  <span className="ml-2 font-mono text-sm font-normal text-subtle">({posts.length})</span>
</h2>
```

---

### 1.3 底部 CTA 区块重复 [P1]

**现状**：页面底部两张 CTA 卡片（"从最近一篇开始" / "找工具和资料"）与 Hero 区域的两个按钮完全重复，无新信息。

**改法**：将底部 CTA 替换为"关于这个站"简介 + 一个 RSS/订阅入口（或直接删除，让页面在 quick-access 区域自然结束）。

## 二、文章详情页（`src/app/posts/[...slug]/page.tsx`）

### 2.1 目录（TOC）不跟踪阅读位置 [P0]

**现状**：TOC 是静态链接列表，无高亮当前章节，读者无法感知自己在文章中的位置。

**对标**：overreacted.io / leerob.io 的 TOC 会高亮当前可见标题。

**改法**：将 TOC 提取为 Client Component，用 `IntersectionObserver` 跟踪 heading 可见性：

```
新建 src/app/components/ui/TableOfContents.tsx（client component）
接收 headings: { id: string; text: string; level: number }[]
内部用 useEffect + IntersectionObserver 监听各 heading 元素
activeId state 控制高亮样式
```

文章页布局同步改为两栏（内容 + 右侧 sticky TOC），仅在 `lg` 断点以上展示：

```tsx
// posts/[...slug]/page.tsx
// 将 article 改为两栏 grid：
<div className="mx-auto max-w-5xl pb-10 lg:grid lg:grid-cols-[minmax(0,1fr)_220px] lg:gap-10">
  <article>...</article>
  {headings.length >= 4 && (
    <aside className="hidden lg:block">
      <TableOfContents headings={headings} />
    </aside>
  )}
</div>
// 同时删除原来内联的 <nav> TOC 块（约第 100-115 行）
```

---

### 2.2 文章页缺少阅读进度条 [P1]

**现状**：长文无任何进度反馈。

**改法**：在 Header 下方加一条 2px 高的进度条（fixed，跟随滚动）：

```
新建 src/app/components/ui/ReadingProgress.tsx（client component）
监听 window scroll，计算 scrollY / (documentHeight - viewportHeight)
用 CSS width 动画驱动，颜色用 var(--accent)
在 posts/[...slug]/page.tsx 顶部渲染
```

---

### 2.3 相关文章展示太弱 [P1]

**现状**：相关内容只是一个简单链接列表，无日期、无描述，视觉权重极低。

**改法**：复用现有 `<PostCard>` 组件替换当前 `<Link>` 列表：

```tsx
// 约第 164-179 行，将 <Link> 替换为：
{relatedPosts.map((related) => (
  <PostCard
    key={related.slug}
    title={related.title}
    description={related.description}
    date={related.date}
    href={`/posts/${related.slug}`}
  />
))}
```

---

### 2.4 文章元数据区域视觉噪音 [P2]

**现状**：kind badge、status badge、日期、修订日期、阅读时间、category、series、tags 全部平铺，信息层级不清晰。

**改法**：
- 第一行（PageHero eyebrow）：只保留 kind + date + 阅读时间
- 第二行（meta bar）：category / series / tags
- status badge 仅在非 published 时显示（已实现，保持）
- tags 超过 4 个时截断并加 `+N` 提示

## 三、博客归档页（`src/app/blog/page.tsx`）

### 3.1 精选入口与归档列表视觉割裂 [P1]

**现状**：精选卡片和下方年份列表是两套完全不同的视觉语言，缺乏整体感。

**改法**：精选卡片改用与归档列表相同的 `PostCard` 组件，保持视觉一致性；或将精选区域改为横向滚动的 featured strip（移动端友好）。

<p className="text-sm text-muted">

## 四、导航页（`src/app/navigation/page.tsx`）

### 4.1 搜索体验缺少键盘快捷键 [P1]

**现状**：搜索框需要鼠标点击才能聚焦，无快捷键。

**对标**：paco.me / rauno.me 的搜索支持 `Cmd+K` 或 `/` 快捷键聚焦。

**改法**：在 `NavigationDirectory.tsx` 中添加：

```tsx
useEffect(() => {
  function handleKey(e: KeyboardEvent) {
    if (e.key === '/' && !['INPUT', 'TEXTAREA'].includes((e.target as Element).tagName)) {
      e.preventDefault();
      inputRef.current?.focus();
    }
  }
  document.addEventListener('keydown', handleKey);
  return () => document.removeEventListener('keydown', handleKey);
}, []);
```

搜索框 placeholder 改为 `搜索链接... (按 / 聚焦)`。

---

### 4.2 链接卡片缺少 favicon [P1]

**现状**：导航链接只有文字和描述，无视觉标识，扫描成本高。

**对标**：所有优秀导航站（rauno.me bookmarks、paco.me tools）都展示 favicon。

**改法**：使用 Google Favicon API（无需后端）：

```tsx
// src/app/components/navigation/CategoryCard.tsx
// 在链接标题前加：
<img
  src={`https://www.google.com/s2/favicons?domain=${new URL(tool.url).hostname}&sz=16`}
  alt=""
  width={16}
  height={16}
  className="h-4 w-4 rounded-sm"
  onError={(e) => { (e.target as HTMLImageElement).style.display = 'none'; }}
/>
```

## 五、Header（`src/app/components/header/`）

### 5.1 当前页面无高亮状态 [P0]

**现状**：导航链接（导航 / 博客）无 active 状态，访客无法感知当前位置。

**改法**：将 `Navigation.tsx` 改为 Client Component，用 `usePathname()` 判断 active：

```tsx
'use client';
import { usePathname } from 'next/navigation';

// 在 Link className 中：
const isActive = pathname.startsWith(item.href);
// active 时加：border-accent-300 bg-accent-50 text-accent
// 替换现有的 border-border bg-surface text-muted
```

---

### 5.2 移动端 Header 过于拥挤 [P1]

**现状**：移动端 Header 同时显示 compact CommandInput + StatusIndicator + 两个导航按钮，空间紧张。

**改法**：移动端隐藏 StatusIndicator（仅 `sm:` 以上显示），导航按钮在移动端改为 icon-only（隐藏文字）：

```tsx
// Navigation.tsx
<span className="hidden sm:inline">{item.text}</span>
```

## 六、性能与 SEO

### 6.1 缺少 OG Image [P0]

**现状**：`generateMetadata` 只设置 title/description，无 `openGraph.images`，分享到社交媒体时无预览图。

**改法**：使用 Next.js `ImageResponse` 生成动态 OG 图：

```
新建 src/app/og/route.tsx
接收 ?title=&description= 参数
返回 ImageResponse（白底 + 标题 + 站点名）
在 generateMetadata 中加：
  openGraph: {
    images: [`/og?title=${encodeURIComponent(post.meta.title)}`]
  }
```

---

### 6.2 缺少 sitemap.xml [P1]

**现状**：无 sitemap，搜索引擎爬取效率低。

**改法**：

```
新建 src/app/sitemap.ts
export default async function sitemap(): Promise<MetadataRoute.Sitemap>
遍历所有 posts，生成 /posts/[slug] 条目
加上 /blog、/navigation 静态页
```

---

### 6.3 缺少 robots.txt [P1]

```
新建 src/app/robots.ts
Disallow: /editor（已有 auth，但 robots 也应屏蔽）
Allow: /
Sitemap: https://yourdomain.com/sitemap.xml
```

---

### 6.4 字体加载策略 [P2]

**现状**：`layout.tsx` 同时加载 3 个 Google Fonts（JetBrains Mono + IBM Plex Sans + Source Serif 4），首屏字体请求多。

**改法**：
- `display: 'swap'` 已隐式设置（Next.js 默认），保持
- 考虑将 Source Serif 4 仅在文章页按需加载（通过 `layout.tsx` 的 segment-level font loading）
- 或将 serif 字体改为系统字体栈 fallback，仅在文章正文使用

## 七、暗色模式 [P2]

**现状**：设计 token 完整（`design-tokens.css` 有完整 warm palette），但无暗色模式切换。

**对标**：antfu.me / leerob.io 均支持暗色模式，且是个人博客的标配。

**改法**：
1. 在 `design-tokens.css` 中添加 `@media (prefers-color-scheme: dark)` 块，覆盖 semantic tokens（`--bg`、`--surface`、`--fg` 等）
2. 在 Header 加主题切换按钮（`useLocalStorage` 存储偏好，`document.documentElement.classList` 切换 `dark` class）
3. Tailwind 配置加 `darkMode: 'class'`

最小实现（仅系统跟随，无手动切换）：只需步骤 1，约 30 行 CSS。
