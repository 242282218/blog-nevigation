# 前端体验优化设计方案

> 生成时间：2026-06-06
> 范围：视觉一致性 / 设计 token / 交互态 / 暗色模式 / 移动端 / 编辑器工作流 / Markdown 预览 / 可访问性
> 原则：项目已用 CSS 变量做主题，优化在 token 层修正而非全站加 dark: 类；保持 Markdown 编辑（不换 WYSIWYG）
> 配套：[项目全量审查报告](./项目全量审查报告.md)、[分阶段优化路线图](./分阶段优化路线图.md)

## 一、前端现状评估（先肯定，再优化）

### 1.1 已建立的优势（避免后续误删）
| 领域 | 说明 |
|------|------|
| 设计系统 | 全站基于 `design-tokens.css` + Tailwind 语义化 token（颜色/间距/圆角/阴影/动效统一变量），无散落硬编码色值 |
| 可访问性基线 | skip-link（`layout.tsx:56`）、aria-label/current/pressed、aria-live 播报、focus-visible 全局描边、prefers-reduced-motion、44px 触控目标 |
| 编辑器工程化 | 质量检查阻断发布、localStorage 草稿自恢复（900ms 防抖）、revision 乐观锁冲突检测、写作/分栏/预览三模式、快捷键（Cmd+S/B/I/K） |
| 渐进增强 | 复制代码 fallback `execCommand`、远程加载失败回退本地、rehype-sanitize + skipHtml 防 XSS、搜索 AbortController 防竞态 |
| 内容架构 | 按年份归档 + 类型/分类筛选 + 精选 + 相关文章 + 全站搜索（文章+链接统一入口），SSR + 动态 OG |

> 这些远高于个人博客平均水平，优化目标是「补短板」而非「重做」。

### 1.2 核心短板（按严重度）
1. **暗色模式可靠性**（P2-1，中）：仅 `prefers-color-scheme` 自动、无手动切换，组件硬编码 `accent-50`/`text-white`，深色对比度未验证。
2. **移动端长文导航缺失**（P2-2，高）：TOC 与归档年份侧栏在 `lg` 以下完全 `hidden`。
3. **编辑器无图片上传**（P2-3，中）：仅插入外链 URL。
4. **本地草稿与服务端同步健壮性**（P2-4，高）：冲突仅提示无 diff 合并，无版本历史/回收站。
5. **键盘可达性细节**（低）：搜索结果/导航列表无 roving tabindex，dialog 无 focus trap。

## 二、设计 token 与视觉一致性优化

### 2.1 发现的不一致（design_consistency）

| 问题 | 位置 | 优化 |
|------|------|------|
| 按钮最小高度不统一 | blog `min-h-[44px]`、editor `min-h-11`/`sm:min-h-9`、header `min-h-[44px]` | 统一 `min-h-11`（44px）为基准，`sm:min-h-9`（36px）仅桌面端次要操作 |
| 卡片悬停效果不统一 | PostCard/CategoryCard/ArticleCard 各自 `-translate-y-0.5` | 抽 `.card-hover` token：transform + shadow + border-color 一致 |
| loading 状态不统一 | editor「处理中…」、blog disabled、CommandInput `animate-pulse`、PreviewPane skeleton | 创建统一 `<LoadingState>` 组件（spinner + 文本） |
| 空状态不统一 | EmptyState dashed border、部分页面纯文本、editor recent 实心 border | 统一用 `<EmptyState>`（icon + title + description + action） |
| 表单错误位置不统一 | FrontmatterForm 字段下方、EditorLoginForm 独立 StatusMessage、settings 字段下方 | field-level 用字段下方文本，form-level 用 StatusMessage |
| badge/pill 变体过多 | `rounded-token-badge`/`rounded-token-full` + 任意颜色组合 | 创建 `<Badge variant="primary|secondary|outline">` 收敛 |
| 字号混用 | 部分 `text-sm`，部分 `text-token-sm` | 优先 design token，token 不覆盖才用原生 |

### 2.2 落地动作
- **新增组件**：`src/app/components/ui/{LoadingState,EmptyState,Badge}.tsx`（EmptyState 可能已存在，统一接口）。
- **token 补充**：`design-tokens.css` 增 `--card-hover-*` 变量。
- **验证**：架构测试可加「按钮 min-h 一致性」断言；视觉回归（可选 Playwright 截图对比）。
- **工作量**：1.5-2 天。

## 三、暗色模式优化（P2-1，重点）

### 3.1 问题
- 仅 `prefers-color-scheme` 自动切换，**用户无法手动选亮/暗/跟随系统**。
- 组件层大量硬编码 `accent-50`、`text-white`（`NavigationDirectory.tsx:92`）、`::selection` 用 terracotta-300（`globals.css:22`），深色下对比度未经验证，存在「半亮半暗」风险。

### 3.2 方案（在 token 层修，不全站加 dark:）
> ⚠️ **不建议**为暗色全面铺设 Tailwind `dark:` 双套类（UX 对比 agent 明确指出会双倍维护成本）。项目已用 CSS 变量，正确做法是：

1. **手动主题切换**：引入 `next-themes`（参考所有现代博客），支持 light/dark/system 三态，存 localStorage + SSR 无闪烁（`suppressHydrationWarning` + 内联脚本）。
2. **token 暗色块修正**：在 `design-tokens.css` 的 `[data-theme="dark"]`（或 `.dark`）块内**修正 token 值**，而非组件加前缀。
3. **消除组件硬编码色**：扫描并替换 `accent-50`/`text-white` 等硬编码为暗色感知 token（如 `text-token-primary`）。
4. **对比度审计**：用 axe DevTools / Lighthouse 验证暗色下所有文本 ≥ WCAG AA 4.5:1（大字 3:1）；重点检查 `NavigationDirectory.tsx:92` 与 `::selection`。

### 3.3 落地
- **涉及**：`src/app/layout.tsx`（ThemeProvider）、`globals.css`/`design-tokens.css`（暗色 token）、组件硬编码色替换、新增主题切换按钮（Header）。
- **验证**：暗色下 axe 无对比度违规；切换无闪烁；刷新保持选择。
- **工作量**：2-3 天。

- **工作量**：移动端整体 2-3 天。

## 五、编辑器工作流优化（P2-3 / P2-4，重点）

### 5.1 图片上传（P2-3，参考 Outstatic）
- **现状**：`MarkdownEditor.tsx` 仅能插入外链 URL，无上传。
- **方案**：复用已集成的 `@aws-sdk/client-s3`：
  1. 拖拽 / 粘贴 / 点击上传图片 → POST `/api/data/upload`（新路由，editor 鉴权）；
  2. 上传到 R2（独立 `images/` 前缀）→ 返回 URL；
  3. 自动插入 `![alt](url)` 到光标位置；
  4. 可选：上传前客户端压缩、Alt 文本编辑、裁剪。
- **涉及**：新增 `src/app/api/data/upload/route.ts`、`MarkdownEditor.tsx` 拖拽处理。
- **验证**：E2E 上传图片 → 预览/公开页正确显示。
- **工作量**：2-3 天。

### 5.2 本地草稿与服务端同步健壮性（P2-4，高）
> ⚠️ **纠正常见误读**：权威数据在服务端 `articles.json`（架构数据流证明），`useLocalArticles` 是**本地草稿/乐观更新缓存层**，不是唯一存储。优化目标是**同步健壮性**而非「迁移存储」。

- **现状问题**：冲突（409）仅提示无 diff 合并；无版本历史/回收站；保存为本地草稿，未自动同步服务器。
- **方案**：
  1. **冲突 diff 合并**：409 时拉取服务端最新版，展示三方 diff（本地/服务端/基线），用户选择合并；
  2. **版本快照**：服务端保存时保留最近 N 个版本（复用 [数据安全方案](./数据安全与备份恢复优化方案.md) 的快照机制），支持回滚；
  3. **回收站软删除**：删除文章移入回收站（保留 30 天）而非硬删；
  4. **自动同步指示**：编辑器顶部「草稿已保存于 X 秒前」「已同步到服务器」状态。
- **涉及**：`useLocalArticles.ts`、`NewArticleContent.tsx`、`editor-data-storage.ts`（版本/回收站）。
- **验证**：清缓存/换设备文章不丢；误删可恢复；冲突可合并。
- **工作量**：3-4 天。

### 5.3 后台操作效率（efficiency_improvements）
| 功能 | 优先级 | 位置 |
|------|--------|------|
| 批量操作（多选 checkbox 批量删/发布/导出） | 高 | `editor/blog/page.tsx` |
| 快捷键帮助面板（Cmd+K / ?） | 中 | 所有 editor 页 |
| 自动保存指示（草稿已保存于 X 秒前） | 中 | `NewArticleContent.tsx` |
| Markdown snippet 库（callout/table/code 模板快速插入） | 中 | `MarkdownEditor.tsx` |
| 文章版本历史（最近 10 版可回滚） | 中 | `NewArticleContent.tsx` |

## 七、可访问性改进（WCAG 2.1 AA）

### 7.1 需修复（accessibility_issues）
| 问题 | 位置 | WCAG | 严重 |
|------|------|------|------|
| 分类按钮缺 `aria-pressed` 标记选中 | `NavigationDirectory.tsx:108` | 4.1.2 | 高 |
| 暗色 `::selection` 对比度需验证 | `globals.css:22` | 1.4.3 | 高 |
| 筛选 Link 缺 aria-label | `blog/page.tsx:150` | 2.4.4 | 中 |
| 密码框无显示/隐藏切换 | `EditorLoginForm.tsx:228` | 3.3.2 | 中 |
| 质量检查按钮 `sm:min-h-9`(36px) < 44px | `NewArticleContent.tsx:613` | 2.5.5(AAA) | 中 |
| description 缺 `role=note` | `posts/[...slug]/page.tsx:104` | 1.3.1 | 低 |

### 7.2 键盘可达性（低但应做）
- 搜索结果列表 + 导航卡片列表加 **roving tabindex**（方向键选择）；
- dialog 加 **focus trap**（防 Tab 出弹窗）；
- 确认现有 `role=toolbar` 按钮的 `aria-keyshortcuts`（已部分实现）。

### 7.3 已正确实现（勿动）
- skip link（`layout.tsx:56`）、CategoryCard favicon `aria-hidden`（`CategoryCard.tsx:49`）、toolbar `role=toolbar`（`MarkdownEditor.tsx:239`）。

## 九、优先级路线图

### 第一阶段（止血，高优先）
- 移动端长文导航（floating TOC + 年份 jump）—— P2-2
- 暗色手动切换 + 对比度审计 —— P2-1 + 7.1 高优先项
- aria-pressed / 触摸目标 44px / 密码显示切换 —— 可访问性高优先
- **验证**：移动端可跳章节；暗色 WCAG AA；axe 无高优先违规

### 第二阶段（补齐核心写作）
- 图片上传（R2）—— P2-3
- 本地草稿同步健壮性（diff 合并 + 版本 + 回收站）—— P2-4
- 预览与编辑滚动联动；设置页字段分组折叠 + 实时预览
- **验证**：可上传插图；清缓存不丢；移动端写作不拥挤

### 第三阶段（一致性打磨）
- 统一 LoadingState/EmptyState/Badge 组件 + token 收敛
- 批量操作 + 快捷键面板 + 自动保存指示
- 代码高亮迁移 rehype-prism-plus
- **验证**：视觉一致性断言通过；后台操作效率提升

### 第四阶段（平台化，按需）
- 评论（Giscus）、上一篇/下一篇、导航收藏/排序、JSON-LD + RSS
- **验证**：内容可互动；SEO 完善

## 十、不建议照搬（避免过度工程）
| 项 | 原因 |
|----|------|
| 富文本 WYSIWYG 替换 Markdown textarea | 当前 Markdown + 实时预览 + 工具栏契合开发者习惯且与文件存储天然对齐，换 WYSIWYG 增加序列化复杂度并破坏 MD 可移植性 |
| 暗色全站铺 `dark:` 双套类 | 项目已用 CSS 变量，应在 token 暗色块修正值，全站加前缀双倍维护成本 |
| 重型状态管理（Redux）/ 实时协作框架（CRDT） | 单人场景数据规模小并发低，现有 hook + 乐观锁已够，属为不存在场景设计 |
| 服务端全文检索引擎（ES/向量） | 链接与文章量级小，前端 includes + 现有 `/api/search` 足够 |

---

## 十一、成功标准
- 移动端长文可跳章节，编辑器不拥挤；
- 暗色可手动切换且全站对比度 WCAG AA；
- 编辑器支持图片上传，草稿同步无丢失、冲突可合并；
- UI 组件（loading/empty/badge）统一，token 无散落硬编码；
- axe / Lighthouse 可访问性无高优先违规；
- 全程 `npm run lint` + UI smoke 通过。
