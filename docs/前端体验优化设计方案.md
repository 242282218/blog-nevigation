# 前端体验优化设计方案

生成日期：2026-06-07
范围：公开博客、导航站、后台编辑器、设置页、Markdown 预览、移动端、可访问性。

## 视觉与交互问题

### P1：warning 对比度不足

`warning` token 和使用场景在小字号 badge 上对比度不足。证据：`tailwind.config.ts:58`、`src/app/editor/(authenticated)/blog/new/ArticleEditorPanels.tsx:141`、`src/app/editor/(authenticated)/blog/new/NewArticleContent.tsx:751`、`src/app/editor/(authenticated)/blog/new/NewArticleContent.tsx:790`。

方案：

- 将 warning 文本 token 调暗，例如浅色模式用更深 amber/brown 文本。
- 对 badge 使用 `bg-warning-50 text-warning-700` 或直接 `text-fg`。
- 禁止仅通过颜色表达 blocking/warning，同时保留文字和 icon。

验证：

- 增加 token 对比度测试或静态颜色快照。
- UI smoke 截图检查 badge 可读性。

### P1：代码块复制按钮 hover 不可读

`.markdown-code-block__copy:hover` 使用 `background: var(--accent-soft)` + `color: var(--accent)`，浅色和暗色都不稳。证据：`src/app/styles/markdown-preview.css:194`、`src/app/styles/design-tokens.css:95`。

方案：

- hover 使用明确的 foreground/background pair。
- 暗色模式单独定义高对比 pair。
- 给 copy 成功态独立 token，不复用 accent-soft。

验证：

- Story/smoke 中截取代码块 hover 状态。
- 手动检查浅色/深色。

### P1：运行时配置表单错误语义弱

编辑口令不一致时只设置顶部消息，确认输入框没有稳定 `aria-invalid`、`aria-describedby`、错误聚焦。证据：`src/app/editor/(authenticated)/settings/runtime/page.tsx:214`、`src/app/editor/(authenticated)/settings/runtime/page.tsx:60`。

方案：

- 抽 `Field` 组件：label、description、control、error、id、aria 自动绑定。
- 错误提交后聚焦第一个错误字段。
- 顶部 `StatusMessage` 只做摘要，不重复字段错误全文。

参考：Pages CMS form 自动绑定 `aria-describedby` 和 `aria-invalid`，证据：https://github.com/pagescms/pagescms/blob/2933a58412092d390f5763edb319ca3c519d3ae2/components/ui/form.tsx#L107-L123。

验证：

- `tests/app/runtime-config-page.test.tsx`：口令不一致时确认框 `aria-invalid=true`，错误说明 id 被引用，焦点落到确认框。

## 设计 token 优化

### 目标

建立语义色的可访问 pair，而不是任意组合。

推荐 token：

- `--status-warning-bg`
- `--status-warning-fg`
- `--status-warning-border`
- `--status-danger-bg`
- `--status-danger-fg`
- `--status-success-bg`
- `--status-success-fg`
- `--interactive-focus-ring`
- `--interactive-hover-bg`
- `--code-copy-hover-bg`
- `--code-copy-hover-fg`

落地规则：

- badge、alert、button hover 必须使用成对 token。
- 深色模式不能只反转单个颜色。
- 文本对比度低于 4.5:1 的组合不得用于正文、小字号 badge、按钮文字。

涉及文件：

- `src/app/styles/design-tokens.css`
- `tailwind.config.ts`
- `src/app/styles/markdown-preview.css`
- `src/app/components/ui/StatusMessage.tsx` 或同类组件

## 选择态 / focus 态 / hover 态

### 选择态

当前编辑器视图按钮有 `aria-pressed`，但父容器缺少 group 语义。证据：`src/app/editor/(authenticated)/blog/new/ArticleEditorPanels.tsx:50`。

方案：

- 父容器加 `role="group" aria-label="编辑视图"`。
- 选中态使用 icon + text + pressed 样式，不只靠颜色。
- 如果未来改成 tab，使用 `role="tablist"` 和 `aria-selected`，不要混用。

### focus 态

移动端“检查文章信息”会聚焦 `article-frontmatter-panel`，但目标 `tabIndex={-1}` 且 `focus-visible:outline-none`，用户看不到跳转结果。证据：`src/app/editor/(authenticated)/blog/new/NewArticleContent.tsx:627`、`src/app/editor/(authenticated)/blog/new/NewArticleContent.tsx:635`。

方案：

- 优先聚焦首个实际错误字段。
- 如果聚焦 panel，则给 panel 程序化 focus 时加短暂 ring 或 scroll-margin。
- 增加 `data-focus-target`，测试可稳定定位。

验证：`tests/app/new-article-content.test.tsx` 继续验证焦点，同时断言可见 ring class 或首个字段聚焦。

### hover 态

规则：

- hover 不得降低对比度。
- icon-only 按钮 hover 保持 44px 可点击区域。
- hover 和 disabled 的视觉必须可区分。

## Loading / Error / Empty 状态

### Loading

多个 loading 只渲染 `animate-pulse` 或 spinner，缺少读屏语义。证据：`src/app/loading.tsx:3`、`src/app/editor/(authenticated)/settings/runtime/page.tsx:305`、`src/app/editor/(authenticated)/blog/new/NewArticleContent.tsx:529`、`src/app/editor/(authenticated)/blog/components/PreviewPane.tsx:30`。

方案：

- 抽 `LoadingState`：`role="status" aria-live="polite"`。
- 装饰 spinner `aria-hidden="true"`。
- loading 文案靠近触发区域，不只放页面顶部。

### Error

现状：部分页面顶部和字段下重复播报错误。证据：`src/app/editor/(authenticated)/settings/page.tsx:269`、`src/app/editor/(authenticated)/settings/page.tsx:119`、`src/app/editor/(authenticated)/navigation/page.tsx:451`、`src/app/editor/(authenticated)/navigation/page.tsx:478`。

方案：

- 顶部只放摘要和“定位问题”。
- 字段下放具体错误。
- `role="alert"` 只给新增的字段错误或摘要之一，避免重复播报。

### Empty

现有导航/文章空状态已有基础覆盖，后续优化：

- 空状态给出直接操作按钮。
- 不用大段说明文字。
- 空状态按钮保持和正常列表操作一致。

## 移动端布局

### 当前风险

- 元信息面板需要滚动定位，定位后 focus 不明显。
- R2 操作按钮禁用但文案不变，移动端反馈可能离触发点太远。
- 长 URL 和长 revision note 可能撑开布局。证据：`src/app/posts/[...slug]/page.tsx:154`、`src/app/editor/(authenticated)/blog/components/PreviewPane.tsx:133`。

### 推荐布局

借鉴 Outstatic 移动端底部状态栏，证据：https://github.com/avitorio/outstatic/blob/8bdf3edb47ac456d2fc86a9ddea846c8cb110e80/packages/outstatic/src/components/document-settings.tsx#L187-L219。

设计：

- 编辑器底部固定操作栏：保存、状态、检查、预览。
- blocking/warning 计数可点击，定位到首个问题。
- 底部栏不遮挡 textarea，预留 safe area。
- R2 设置页按钮内展示“同步中/恢复中”，并在按钮旁显示状态。

验证：

- Playwright mobile viewport 截图。
- 检查横向溢出。
- 检查底部操作栏按钮触达尺寸。

## 后台编辑工作流

当前项目已有 draft/published/evergreen 和 blocking check。证据：`src/app/editor/(authenticated)/blog/page.tsx:103`、`src/app/editor/(authenticated)/blog/page.tsx:163`、`src/app/editor/(authenticated)/blog/new/NewArticleContent.tsx:236`。

推荐从“文章列表按钮”升级为轻量工作流：

- 草稿：未发布文章。
- 待修复：blocking check 未通过。
- 可发布：blocking check 通过但未发布。
- 已发布：published/evergreen。

借鉴 Decap editorial workflow，但不引入 Git PR/label。证据：https://github.com/decaporg/decap-cms/blob/11f35405cd1a1d1216e97b890173f46436752019/cypress/e2e/common/editorial_workflow.js#L27-L92。

落地：

- 文章列表增加 segmented control 或 tabs。
- 每篇文章展示质量检查摘要。
- 批量发布先只支持“可发布”列表。
- 发布前检查服务端化，避免只依赖客户端。

## Markdown 预览体验

当前编辑器预览目录缺少命名和 focus 样式，公开文章 TOC 更完整。证据：`src/app/editor/(authenticated)/blog/components/PreviewPane.tsx:103`；公开 TOC 对照：`src/app/components/ui/TableOfContents.tsx:60`。

方案：

- 预览目录复用 `TableOfContents` 或抽出 shared TOC。
- `nav` 加 `aria-label="预览目录"`。
- 当前标题加 `aria-current`。
- 链接加 focus-visible ring。
- 长 heading 使用 `overflow-wrap:anywhere`。

Markdown 安全：

- 继续使用 `react-markdown` + `rehype-sanitize`。
- 不照搬 MDX 任意组件执行。
- 如果要支持 callout/math，先加 sanitize schema 和测试。

## 可访问性改进清单

1. warning/hover 对比度修复。
2. `Field` 组件统一 `aria-invalid`、`aria-describedby`。
3. loading 统一 `role=status`。
4. 编辑器视图切换加 group 语义。
5. 移动端错误定位有可见 focus。
6. 预览目录有 `aria-label` 和 focus 样式。
7. R2 忙碌态按钮内反馈。
8. 表单错误避免重复播报。
9. 长文本 `overflow-wrap:anywhere`。
10. UI smoke 加 a11y 断言和移动端截图。
