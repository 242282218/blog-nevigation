## 测试缺口

### P1：Docker 更新缺少行为级测试

现有测试主要是字符串断言，不能证明更新脚本真的保留 `.env` 和 `data/`。证据：`deploy/git-deploy.sh:115`、`tests/architecture/repository-structure.test.ts:173`、`tests/architecture/repository-structure.test.ts:205`。

应补：

- `tests/scripts/docker-update.test.ts`
- 用临时 `DEPLOY_PATH`。
- fake `docker`、`git`、`tar` 放 PATH 前面。
- 执行 `deploy/git-deploy.sh`。
- 断言 `.env` 未被覆盖、`data/` 未被删除、tar 参数包含 `.env data`、compose 仍是 bind mount。

### P1：Docker smoke 没挂载真实数据目录

CI smoke 当前 `docker run` 只传 env，缺少 `-v <temp>:/var/lib/blog-navigation`，不能发现权限、entrypoint、持久化问题。证据：`.github/workflows/docker-deploy.yml:77`。

应补：

- 创建 `.tmp/docker-smoke-data`。
- `docker run -v "$PWD/.tmp/docker-smoke-data:/var/lib/blog-navigation"`。
- 容器启动后写入一个 runtime 文件。
- 重建容器后确认文件仍在。

### P1：R2 外部协议全是 mock

单元覆盖细，但 Cloudflare/S3 真实 Body 类型、403/404/NoSuchKey、权限返回 shape 变化无法发现。

应补：

- 可选本地 S3 兼容服务契约测试。
- 或 nightly 真实 R2 测试，必须使用测试 bucket。
- 覆盖 PutObject/GetObject ContentType、NoSuchKey、超限对象。

### P1：manifest 事务失败注入缺失

当前没有测试“资源写成功但 manifest 写失败”的一致性结果。

应补：

- mock `writeManifest` 抛错。
- 断言不会半提交，或启动恢复 marker 可诊断。
- 测试 articles/navigation/settings 三类资源。

### P2：UI 可访问性缺口

缺颜色对比、runtime 表单错误、loading role、预览目录、移动端可见 focus 测试。

证据：`tailwind.config.ts:58`、`src/app/styles/markdown-preview.css:194`、`src/app/editor/(authenticated)/settings/runtime/page.tsx:214`、`src/app/loading.tsx:3`、`src/app/editor/(authenticated)/blog/components/PreviewPane.tsx:103`。

应补：

- token 对比度测试。
- runtime 表单错误 aria 测试。
- loading `role=status` 测试。
- PreviewPane TOC `aria-label` 和 focus class 测试。
- 移动端错误定位可见 focus 测试。

### P2：公开输出 0 覆盖

`robots.ts`、`sitemap.ts`、`og/route.tsx`、`api/favicon/route.ts` 等公开输出应补轻量测试。

应补：

- URL 正确。
- content-type 正确。
- 缓存头正确。
- site URL 配置错误时有合理 fallback。

## Mock 风险

- `remote-backup-route.test.ts` mock 掉 R2 下载/同步，只验证路由编排，不验证真实上传下载联动。
- `editor-data-routes.test.ts` mock 掉 queue，只验证“调用入队”，不验证 pending 写失败对 API 的影响。
- UI smoke 部分关键列表路径用 route fulfill 和 localStorage，降低服务端持久化合同覆盖。证据：`scripts/test/verify-editor-blog-ui.py:101`、`scripts/test/verify-editor-blog-ui.py:319`。

缓解：

- 保留单元 mock，但补一层契约测试。
- UI smoke 增加“真实 API 模式”和“离线兜底模式”两个路径。
- 对 R2 明文、Docker 保数据这类硬约束，优先行为测试，不只静态断言。

## 应补集成测试

### Docker 更新模拟

测试名：`tests/scripts/docker-update.test.ts`

流程：

1. 创建临时 deploy 目录。
2. 写入 `.env` 和 `data/settings/site.json`。
3. fake `docker` 记录命令但不真实执行。
4. fake `git` 返回固定 commit。
5. fake `tar` 记录打包参数。
6. 执行 deploy 脚本。
7. 断言 `.env` 内容未变。
8. 断言 `data/settings/site.json` 未变。
9. 断言打包包含 `.env data`。
10. 断言没有 `rm -rf data` 行为。

### R2 契约测试

模式：

- 默认跳过。
- 设置 `RUN_R2_CONTRACT_TESTS=true` 时运行。
- 使用测试 bucket 和测试 prefix。
- 测试结束清理测试 prefix。

断言：

- PutObject body 是明文 JSON。
- GetObject 后 parse 成相同 payload。
- 404/NoSuchKey 变成可诊断错误。
- 权限不足不写本地。

### 数据恢复集成

流程：

1. 临时数据根写入文章/导航/设置。
2. 生成 backup payload。
3. 修改本地数据。
4. 恢复旧 payload。
5. 校验恢复前 snapshot 存在。
6. 校验 manifest hash。
7. 校验 R2 latest 入队。

## 应补 Docker smoke

推荐 workflow 步骤：

1. Buildx `load: true` 构建 `blog-navigation:ci-smoke-${sha}`。
2. 创建 `.tmp/docker-smoke-data`。
3. `docker run -v "$PWD/.tmp/docker-smoke-data:/var/lib/blog-navigation"`。
4. health endpoint 通过。
5. 写入一个 marker 文件或通过 API 写入设置。
6. 停止并重启容器。
7. 验证 marker/设置仍在。
8. Trivy 扫描同一镜像。
9. 扫描通过再 push。
