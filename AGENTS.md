# 项目级执行规则

## 备份规则

- R2 远端备份必须保持明文 JSON。
- 不要为 R2 备份新增、恢复或要求任何加密口令字段。
- 不要新增 `R2_BACKUP_ENCRYPTION_PASSPHRASE` 环境变量、UI 输入框、API 字段或文档说明。
- R2 备份上传内容必须能直接 `JSON.parse`，方便检查、迁移和恢复。
- 如需加密备份，只能放在独立导出/导入工具链，不得影响 R2 自动备份链路。

## Docker 更新规则

- 更新 Docker 镜像或容器时必须保留现有数据和配置。
- 不得删除、重建或覆盖生产数据目录，例如 `/opt/blog-nevigation/data`。
- 不得删除或覆盖生产 `.env`，否则会导致用户每次更新后重新配置。
- 更新前必须确认数据目录和 `.env` 是宿主机持久化挂载，不允许改成匿名 volume。
- 更新前必须先做本地快照备份，至少包含 `.env` 和 `data/`。
- 更新脚本和 README 必须明确说明：镜像可替换，数据目录和配置文件必须保留。

## 验证规则

- 涉及 R2 配置、备份、恢复、Docker 更新或部署脚本的改动，必须检查是否违反以上规则。
- 涉及 R2 的测试应覆盖：不保存旧加密字段、上传明文 JSON、下载后可直接恢复。
- 涉及 Docker 更新的文档或脚本应覆盖：更新前备份、保留宿主机数据目录、保留 `.env`。
